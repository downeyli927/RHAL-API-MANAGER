### RHAL_AI_Process
```bash
SINT32 RHAL_AI_Process(void*handle, NN_HW_INDEX hw, AI_CALLBACK_PTR ptr, UINT32 timeout);
```
This function is used to "trigger a processing task for a specific AI hardware stage" and sets parameters such as callback functions and timeouts for that stage.
<ParamTable
  items={[
    { name: 'handle', type: 'void*', desc: 'A pointer to the AI instance returned after initialization' },
    { name: 'hw', type: 'NN_HW_INDEX', desc: <>The index of the hardware stage to be triggered.</> },
    { name: 'ptr', type: 'AI_CALLBACK_PTR', desc: <>A pointer to the callback function, called when the corresponding stage thread completes processing or when an event occurs..</> },
    { name: 'timeout', type: 'UINT32', desc: 'The timeout setting for this task, used as a reference by the stage thread during execution.' },
  ]}

### RHAL_AI_CheckReadyToRun
```bash
SINT32 RHAL_AI_CheckReadyToRun(void*handle, volatile CHAR*pCancel);
```
This function is used to determine whether an AI inference/processing loop can begin at the current time, based on the target frame rate and the internal "next run time" scheduling.


### RHAL_AI_UpdateRunTime
```bash
SINT32 RHAL_AI_UpdateRunTime(void*handle);
```
After an AI processing/inference operation is completed, update the time point for the next allowed run (time2Run) and count the number of runs for the corresponding output function index.
