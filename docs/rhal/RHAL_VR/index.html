RHAL_VR_GetVideoFrameBufferIndex​

DTV_STATUS_T RHAL_VR_GetVideoFrameBufferIndex(HAL_VR_VIDEO_WINDOW_TYPE_T videoWindowID, UINT32*pIndexOfCurrentVideoFrameBuffer);

Get index of video frame buffer corresponding to videoWindowID. The video frame buffer corresponding to current index is same with video frame buffer that last write operation is completed. This buffer may be 1~2 frame delayed buffer compared to video frame being displayed on the screen.

Parameters
videoWindowIDHAL_VR_VIDEO_WINDOW_TYPE_T: video window ID
pIndexOfCurrentVideoFrameBufferUINT32*: index of current video frame buffer

RHAL_VR_GetVideoFrameBufferSecureVideoState​

DTV_STATUS_T RHAL_VR_GetVideoFrameBufferSecureVideoState(HAL_VR_VIDEO_WINDOW_TYPE_T videoWindowID, UINT8*pIsSecureVideo);

Get secure video state of video frame buffer corresponding to videoWindowID. Vdec driver can know SVP state of own. And VT driver can know which vdec is mapped to videoWindowID. Therefore, If VT driver can get vdec's SVP state, VT driver can provide the SVP state information to VT client through this interface.

Parameters
videoWindowIDHAL_VR_VIDEO_WINDOW_TYPE_T: video window ID
pIsSecureVideoUINT8*: state of secure video corresponding to videoWIndowID
